What is HTML?

Day 1 · Topic 01
H
HyperText

Text with links that connect to other pages or resources.

M
Markup

A way to annotate text with special tags to give it meaning.

L
Language

A set of rules and symbols that browsers understand.

HTML is NOT a programming language. It is a markup language — it describes the structure and meaning of content.
🌐

What browsers do

Browsers read HTML files and display them as web pages. The tags are not shown — they tell the browser how to display content.

example.html
<h1>Hello, World!</h1>
<p>This is a paragraph.</p>

HTML Document Structure

Day 1 · Topic 02 — Every HTML file must follow this structure
index.html
<!-- Declaration -->
<!DOCTYPE html>

<!-- Root element -->
<html lang="en">

  <!-- Meta information -->
  <head>
    <meta charset="UTF-8">
    <title>My First Page</title>
  </head>

  <!-- Visible content -->
  <body>
    <h1>Hello, World!</h1>
    <p>Welcome to my page.</p>
  </body>

</html>
1
<!DOCTYPE html>

Tells the browser this is an HTML5 document. Always first!

2
<html>

Root element. Contains the entire page. Add lang="en" for accessibility.

3
<head>

Invisible info: charset, title, links to CSS. Not displayed on the page.

4
<body>

Everything the user sees on the page goes here.

Block vs Inline & <div> / <span>

Day 2 · Topics 03 & 04 — Layout containers

Block-level elements

Start on a new line and take full width. Examples: <div>, <h1>–<h6>, <p>, <ul>, <table>

Inline elements

Flow within text, take only as much width as needed. Examples: <span>, <a>, <strong>, <img>

containers.html
<!-- div: block container for layout -->
<div class="section">
  <h2>About</h2>
  <p>Some text here.</p>
</div>

<!-- span: inline container for styling -->
<p>
  The sky is 
  <span style="color:blue;">
    blue
  </span>
   today.
</p>
<div> has no visual style on its own — it's purely a grouping container. <span> is the same but inline. Both become powerful when combined with CSS classes.

Accessibility Basics

Day 3 · Topic 07 — Making web pages usable for everyone
1
Always use alt text on images

<img src="..." alt="Descriptive text"> — screen readers read this aloud.

2
Use semantic HTML

<header>, <nav>, <main>, <button> are understood by assistive technology.

3
Label all form inputs

<label for="id"> linked to each <input id="id"> is essential.

4
Meaningful link text

Write "Read the report" not "Click here". Screen reader users navigate by links.

What is CSS?

CSS Module · Topic 01 — Cascading Style Sheets
C
Cascading

Styles flow down from parent to child elements, and rules can override each other in a defined order.

S
Style

Controls the visual appearance — colors, fonts, spacing, layout, animations.

S
Sheets

Written in separate .css files (or inside HTML) as a list of rules.

HTML = Structure  |  CSS = Appearance
HTML says "this is a heading". CSS says "make it purple, 32px, bold".
A CSS rule
/* selector { property: value; } */

h1 {
  color: purple;
  font-size: 2rem;
  font-weight: bold;
}

p {
  color: #333;
  line-height: 1.6;
}
A CSS rule has three parts: a selector (what to style), a property (what aspect), and a value (how to style it).

Inline CSS

CSS Module · Topic 02 — Styles applied directly on an element
inline.html
<!-- Style goes in the style="" attribute -->

<h1 style="color: purple; font-size: 2rem;">
  Hello World
</h1>

<p style="color: #555; line-height: 1.8;">
  This paragraph has custom spacing.
</p>

<div style="background: yellow;
            padding: 10px;
            border: 2px solid red;">
  A styled box
</div>
✅

When to use inline CSS

Quick one-off styles, email HTML templates, dynamically set styles via JavaScript, or overriding styles in a pinch.

⚠️ Avoid overusing inline CSS. It mixes structure and style, is hard to maintain, cannot be reused, and has the highest specificity — overriding all other styles.
Syntax: Multiple declarations are separated by semicolons inside the style="" attribute: style="prop1: val1; prop2: val2;"

Internal CSS

CSS Module · Topic 03 — Styles in the <head> using a <style> tag
internal.html
<!DOCTYPE html>
<html lang="en">
<head>
  <title>My Page</title>

  <!-- Internal CSS lives here -->
  <style>
    body {
      font-family: Arial, sans-serif;
      background: #f0f0f0;
    }

    h1 {
      color: #333;
      text-align: center;
    }

    p {
      color: #666;
      font-size: 1.1rem;
    }
  </style>

</head>
<body>
  <h1>Hello!</h1>
  <p>Styled by internal CSS.</p>
</body>
</html>
📄

How internal CSS works

CSS rules are written inside a <style> tag placed in the <head> section. They apply to the entire current page only.

✅

Good for:

Single-page demos, quick prototypes, or pages that need unique styles not shared elsewhere.

Limitation: Styles only affect this one page. If you have 10 pages, you'd need to copy the <style> block into each one — that's hard to maintain.

Step 2: Link it in your HTML <head>
index.html
<head>
  <title>My Site</title>

  <!-- Link external CSS file -->
  <link
    rel="stylesheet"
    href="style.css"
  >

  <!-- Link from subfolder -->
  <link
    rel="stylesheet"
    href="css/style.css"
  >

  <!-- Link from CDN (e.g. Google Fonts) -->
  <link
    rel="stylesheet"
    href="https://fonts.googleapis.com/
css2?family=Roboto"

One CSS file → many HTML pages. Change the style in one place and all pages update instantly. This is the professional standard.

Comparing the 3 Methods

CSS Module · Inline vs Internal vs External — when to use each
MethodWhere writtenScopeBest forRecommended?
Inline style="" attribute on element Single element only One-off overrides, JS-generated styles ⚠️ Use sparingly
Internal <style> tag in <head> Current page only Single-page demos, prototypes 🟡 OK for learning
External Separate .css file, linked via <link> All linked pages Real websites, all professional projects ✅ Always preferred
Inline priority: Overrides everything. Hard to maintain. Avoid for layout.
Internal priority: Overrides external CSS. Good for page-specific tweaks.
External priority: Lowest but cleanest. Browser can cache it — pages load faster!
File structure tip: Keep your CSS in a css/ folder: my-project/css/style.css and link with href="css/style.css"

box-model.css
.box {
  /* Content size */
  width:  200px;
  height: 100px;

  /* Space inside border */
  padding: 20px;

  /* Visible edge */
  border: 2px solid black;

  /* Space outside border */
  margin: 30px;

  /* Include padding in width */
  box-sizing: border-box;
}
box-sizing: border-box — always add this! It makes width include padding + border, so layout math is much simpler.

Specificity & The Cascade

CSS Module · Topic 08 — Why does one style win over another?

🏆 Specificity Score (highest wins)

Selector typeScore
Inline style style=""1,0,0,0
ID selector #id0,1,0,0
Class / attribute / pseudo-class0,0,1,0
Element selector p0,0,0,1
Universal *0,0,0,0
!important overrides ALL specificity — use it only as a last resort. It breaks the cascade and causes maintenance nightmares.
cascade.css
/* Score: 0,0,0,1 — element */
p { color: black; }

/* Score: 0,0,1,0 — class (wins) */
.intro { color: blue; }

/* Score: 0,1,0,0 — ID (wins) */
#hero { color: red; }

/* Score: 1,0,0,0 — inline (wins) */
<p style="color: green;">

/* Same score → last rule wins */
p { color: orange; }
p { color: teal; }
/* ↑ teal wins — written last */
